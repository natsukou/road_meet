# RoadMeet - GPS社交交友线下聚会H5应用 PRD

## 1. 产品概述

### 1.1 产品定位
基于GPS定位的社交交友线下聚会H5应用，帮助两个用户找到彼此中间的有趣地点进行线下见面。

### 1.2 目标用户
年轻人群体，寻求新颖社交方式的用户。

### 1.3 核心价值
- 解决"在哪里见面"的决策难题
- 增加见面的趣味性和期待感
- 记录珍贵的线下社交时刻

---

## 2. 功能需求

### 2.1 用户系统
| 功能 | 描述 |
|------|------|
| 手机号注册 | 输入手机号获取验证码完成注册 |
| 手机号登录 | 输入手机号和验证码登录 |
| 极简资料 | 仅需手机号，不收集其他用户信息 |

### 2.2 见面任务系统
| 功能 | 描述 |
|------|------|
| 创建见面任务 | 生成唯一的16进制见面代号（如：#A3F9B2） |
| 匹配见面任务 | 通过输入昵称、代号或16进制代号匹配对方 |
| 编辑限制 | 每人最多编辑6次任务信息 |
| 取消机制 | 一方取消时通知另一方，弹窗确认后双方同意才删除 |
| 删除机制 | 可删除自己创建的未匹配任务 |

### 2.3 GPS与地点推荐
| 功能 | 描述 |
|------|------|
| 获取双方位置 | 调用浏览器GPS接口获取双方实时位置 |
| 中间点计算 | 计算两人位置的地理中点 |
| 地点推荐 | 调用高德/阿里云地图API推荐中点周边的有趣地点 |
| 兴趣标签 | 结合兴趣标签（咖啡、餐厅、公园、书店等）筛选地点 |

### 2.4 见面记录
| 功能 | 描述 |
|------|------|
| 见面打卡 | 到达地点后可打卡确认见面 |
| 过程记录 | 可选上传见面照片、文字记录 |
| 历史查看 | 查看过往见面记录 |

## 3. 技术方案

### 3.1 技术栈
- **框架**: React + Vite
- **样式**: Tailwind CSS
- **地图**: 高德地图JS API
- **状态管理**: React Context / useState

### 3.2 数据策略
- 使用浏览器LocalStorage模拟数据存储
- 无需后端服务，纯前端实现
- 无需部署，本地运行即可

## 7. 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| GPS权限拒绝 | 提示手动输入位置或选择地点 |
| 一方离线 | 显示离线状态，保留任务 |
| 编辑次数用尽 | 提示无法编辑，只能取消 |
| 取消确认超时 | 24小时后自动取消 |
| 代号不存在 | 提示检查代号是否正确 |
| 网络异常 | 本地缓存，恢复后同步 |
